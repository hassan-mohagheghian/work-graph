# ADR 001: Domain-Driven Design

## Status

Accepted

## Context

WorkGraph has multiple bounded contexts: Identity, Workspace, Project, Knowledge, AI.

## Decision

Use DDD principles to separate modules.  
Each domain has `domain`, `application`, `infrastructure`, `presentation`.

## Alternatives Considered

- Monolithic CRUD → easier to start, harder to scale

## Consequences

- Clear boundaries
- Easy to transition to microservices
- Increased initial complexity
