# 004 - 30 Day Challenge Plan

## Overview

This document defines the scope for the 30-Day Build Challenge.

The objective is not to build WorkGraph from scratch.

The objective is to transform the current foundation into a usable product that demonstrates the core value proposition of WorkGraph v2.

Core Product Flow:

Goal
↓
Documents
↓
Roadmap
↓
Milestones
↓
Tasks

Success means delivering a working end-to-end workflow by the end of the challenge.

---

# Week 1 — Knowledge Foundation

## Goal

Introduce the Knowledge domain and connect it to projects.

## Deliverables

### Backend

* Document entity
* Document repository
* Document CRUD APIs
* Project-to-document relationships

### Frontend

* Documents page
* Create document
* Edit document
* Delete document
* Document listing

### Outcome

Users can store project knowledge inside WorkGraph.

Projects become more than task containers.

Knowledge becomes part of the workflow.

---

# Week 2 — AI Planning Engine

## Goal

Transform documents into structured project plans.

## Deliverables

### AI Capabilities

* Goal extraction
* Requirement extraction
* Document summarization

### Planning Capabilities

Generate:

* Goals
* Roadmap
* Milestones

### User Experience

User uploads:

* Product vision
* Requirements
* Meeting notes

WorkGraph returns:

* Suggested goals
* Suggested roadmap
* Suggested milestones

### Outcome

WorkGraph begins transforming knowledge into plans.

---

# Week 3 — AI Task Generation

## Goal

Convert planning artifacts into executable work.

## Deliverables

### AI Capabilities

Generate tasks from:

* Goals
* Roadmaps
* Milestones

### Project Integration

* Create tasks from AI output
* Assign priorities
* Link tasks to milestones

### User Experience

User reviews generated tasks before creation.

### Outcome

WorkGraph can transform documentation into actionable work.

Goal
↓
Roadmap
↓
Milestones
↓
Tasks

is now operational.

# Week 4 — Polish, Search & Demo

## Goal

Prepare a complete end-to-end demonstration.

## Deliverables

### Knowledge Search

* AI-assisted search

### Product Improvements

* UX improvements

* Error handling

1. Create project
2. Create document
3. Generate goals

### Outcome

WorkGraph demonstrates its complete value proposition.

Knowledge
↓
Planning
↓

---

# Challenge Success Criteria

By Day 30, a user can:

* Create an organization
* Create a project
* Write project documentation
* Generate goals from documentation
* Generate a roadmap
* Generate milestones
* Generate tasks
* Manage generated tasks

The system demonstrates the WorkGraph v2 vision:

Transform goals and knowledge into execution.
